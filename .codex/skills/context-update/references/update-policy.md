# Per-File Update Policy

- `progress.md`: always update.
- `project-structure.md`: update when directories/files move or are added.
- `tech-context.md`: update when dependencies/tooling change.
- `system-patterns.md`: update when architecture patterns change.
- `product-context.md`: update when requirements/user flows change.
- `project-brief.md`: update only for goal/scope shifts.
- `project-overview.md`: update at milestone-level capability changes.
- `project-vision.md`: update only for strategic direction changes.
- `project-style-guide.md`: update when standards/conventions change.
